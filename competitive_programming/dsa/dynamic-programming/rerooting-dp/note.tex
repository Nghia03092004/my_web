\section*{Overview}

Rerooting DP is the pattern for ``compute the same tree answer for every possible root.'' Instead of rebuilding the tree
DP from scratch \(n\) times, it uses one pass for subtree information and one pass for the contribution coming from the
parent side.

\section*{When to Use It}

Use it when:

\begin{itemize}
  \item the answer is wanted at every vertex,
  \item the rooted answer can be decomposed into child contributions plus one parent-side contribution,
  \item moving the root across one edge can be described locally.
\end{itemize}

\section*{Core Idea}

Do two traversals:

\begin{itemize}
  \item a downward pass computing ordinary subtree DP,
  \item an upward pass telling each child what the rest of the tree contributes once that child becomes the root.
\end{itemize}

\section*{Key Insight}

The hard part is not the second DFS itself. It is designing a state that can be transferred across an edge after removing
the child's own contribution and adding the parent-side contribution.

\section*{Worked Problem}

\paragraph{Problem.}
For every vertex \(v\) in a tree, compute the sum of distances from \(v\) to all other vertices.

\paragraph{First pass.}
For each node:

\begin{itemize}
  \item \texttt{sub\_size[v]} = size of the subtree,
  \item \texttt{down[v]} = sum of distances from \(v\) to nodes in its subtree.
\end{itemize}

\paragraph{Transfer formula.}
If \(to\) is a child of \(v\), then rerooting from \(v\) to \(to\) changes the answer by:

\begin{itemize}
  \item subtract \(\texttt{sub\_size[to]}\), because those nodes become one step closer,
  \item add \(n - \texttt{sub\_size[to]}\), because all other nodes become one step farther.
\end{itemize}

So:
\[
ans[to] = ans[v] - sub\_size[to] + (n - sub\_size[to]).
\]

\section*{Correctness Intuition}

The first pass is ordinary tree DP. The second pass is correct because every node outside the child's subtree changes
distance by \(+1\), and every node inside changes by \(-1\). Those are the only two cases.

\section*{Complexity Analysis}

When each edge transfer is \(O(1)\), rerooting runs in \(O(n)\).

\section*{Implementation}

The code solves the sum-of-distances example, which is a good baseline because the transfer formula is easy to inspect
and verify.

\section*{Common Pitfalls}

\begin{itemize}
  \item Passing a child its own contribution again by mistake.
  \item Designing a state that is enough for one root but not enough to transfer.
  \item Recomputing sibling merges from scratch instead of deriving them with prefix/suffix or a direct formula.
\end{itemize}

\section*{Variants / Extensions}

\begin{itemize}
  \item Maximum or minimum score over all roots.
  \item Prefix-suffix rerooting when a child needs the merge of all other children.
  \item Generic rerooting frameworks with custom combine and lift operators.
\end{itemize}

\section*{Practice Problems}

\begin{itemize}
  \item Sum of distances from every node.
  \item Best root under a subtree-based score.
  \item Count rooted structures for every root.
\end{itemize}
